# 赛题三：叙事剧情游戏 — AI 重塑叙事 Ideas

> **背景：** 腾讯黑客松，CodeBuddy Agent 辅助，3人团队
> **引擎：** Godot / Unity / 纯 Web 前端（AI 对话类推荐纯前端方案）
> **核心理念：** 不仅是"用 AI 做游戏"，更是"做 AI 驱动的游戏"，与黑客松的技术主题高度契合。

---

## Idea 1：《巷口杂货铺》— AI 角色驱动

| 属性 | 评估 |
|------|------|
| **类型** | 经营模拟 + AI 自由对话 |
| **难度** | ★★★★☆（中高） |
| **CodeBuddy 适配度** | ★★★★★ — AI API 调用封装、对话状态管理、NPC 人格系统，AI 全流程覆盖 |
| **3人团队可行性** | ★★★★☆ — 1人对接 AI API + Prompt + 后端，1人前端/UI，1人经营逻辑+叙事设计 |
| **独特性** | ★★★★★ — AI NPC 自由对话 + 经营模拟，市面上尚无类似产品 |

**玩法：**
- 你继承了一家老社区的杂货铺，每天接待不同的 AI 邻居顾客
- 每个 NPC 由 AI 驱动，有自己的性格、背景故事和当日心情。玩家通过自由对话了解他们的需求，推荐商品、倾听烦恼、甚至调解邻里矛盾
- 杂货铺的经营决策（进什么货、是否赊账）会影响社区关系和店铺收益
- **AI 亮点：** AI NPC 每次对话动态生成，游戏永不重复；可以自定义 NPC 的"世界观设定书"（AI 的人格文档）
- **开发要点：** 核心是 Prompt 工程，前端交互不复杂，建议 Web 方案

## Idea 2：《轮回戏班》— AI 生成剧情

| 属性 | 评估 |
|------|------|
| **类型** | 角色扮演 + AI 剧情生成 |
| **难度** | ★★★★☆（中高） |
| **CodeBuddy 适配度** | ★★★★★ — 故事生成引擎、选择分支状态管理、剧情一致性校验，AI 高度契合 |
| **3人团队可行性** | ★★★★☆ — 纯文本+选择分支，1人专攻 AI Story Engine + Prompt，1人前端，1人叙事/测试 |
| **独特性** | ★★★★★ — AI 实时生成戏曲题材叙事，市场上没有对标产品 |

**玩法：**
- 你是一个即将解散的传统戏班的班主，需要在最后 7 天里找到接班人
- 每天选择去不同地点（茶馆、祠堂、戏台、码头），AI 生成该场景的剧情和角色
- 每个选择影响戏班命运：找到徒弟、获得演出机会、面临债主逼门…
- 游戏的"剧本"完全由 AI 实时生成，每个玩家都是不同的故事
- **AI 亮点：** AI 作为"故事引擎"，根据玩家的选择习惯动态调整剧情走向
- **开发要点：** 纯文本驱动 + 选择分支，前端极简，核心在 prompt 设计和剧情一致性

## Idea 3：《旧物回收站》— AI 记忆重构

| 属性 | 评估 |
|------|------|
| **类型** | 解谜 + AI 推理叙事 |
| **难度** | ★★★★☆（中高） |
| **CodeBuddy 适配度** | ★★★★★ — 物品组合推理逻辑、AI 叙事生成、线索管理，AI 能覆盖大部分代码 |
| **3人团队可行性** | ★★★★☆ — 核心是 AI 推理逻辑，前端交互简单，适合技术导向的团队 |
| **独特性** | ★★★★★ — AI 推理+情感叙事的混搭，概念新颖且有深度 |

**玩法：**
- 你是一个旧物回收站的店员，收到一批遗物，需要通过物品推断逝者的故事，找到遗愿清单并完成它
- 玩家检查物品（老照片、信件、怀表等），AI 根据物品组合生成推测叙事，玩家逐步拼凑出完整故事
- 找到遗愿后，通过决策去完成（如把信送到故人手中、把一个勋章归还战友后代）
- **AI 亮点：** AI 驱动的"推理引擎"— 不同玩家发现不同线索会拼出不同的故事版本
- **开发要点：** 点选式交互 + AI 文本生成，重点在叙事设计和 prompt 工程

## Idea 4：《面试官》— AI 人格模拟

| 属性 | 评估 |
|------|------|
| **类型** | AI 对话模拟 + 策略判断 |
| **难度** | ★★★☆☆（中） |
| **CodeBuddy 适配度** | ★★★★★ — 纯对话系统，几乎没有传统"游戏编程"，AI 全栈都能写 |
| **3人团队可行性** | ★★★★★ — 几乎不需要美术资源，2人写代码+Prompt，1人设计面试题库和角色 |
| **独特性** | ★★★★★ — 职场面试+AI 人格模拟，社交传播潜力极大（面试梗天然适合分享） |

**玩法：**
- 你是一家公司的 HR，需要面试 5 个 AI 驱动的候选人竞争同一个岗位
- 每个候选人由 AI 驱动，拥有独特的简历、性格、优点和隐藏缺点。玩家自由提问，AI 候选人会基于其设定动态回答
- 面试结束后，玩家必须决定录用谁。游戏结局取决于玩家的判断 — 选对人公司腾飞，选错人被坑
- **AI 亮点：** 同一职位，每次重玩 AI 候选人的回答都有变化；可以设定不同职位的"面试题库"
- **开发要点：** 纯对话界面，无任何物理/动画系统，Web 应用即可，重点在角色 Prompt 设计

## Idea 5：《人生蒙太奇》— AI 人生模拟

| 属性 | 评估 |
|------|------|
| **类型** | 卡片叙事 + AI 生成 |
| **难度** | ★★★☆☆（中） |
| **CodeBuddy 适配度** | ★★★★★ — 卡片交互逻辑、AI 生成流式输出、记忆拼接算法，AI 全覆盖 |
| **3人团队可行性** | ★★★★★ — 卡片 UI+AI 调用，纯前端项目，3人高效产出 |
| **独特性** | ★★★★★ — "用碎片拼接人生故事"的概念非常新颖，艺术感强 |

**玩法：**
- 你是一台"人生放映机"的操作员，通过拼接不同人的记忆片段来讲述他们的一生
- 玩家从"记忆库"中选择碎片（童年、初恋、失业、旅行…），AI 根据选择的碎片组合生成一段连贯的人生故事
- 玩家的选择决定故事的基调（喜剧/悲剧/温情）和走向
- 目标是给每个 NPC 的人生剪辑出最精彩的一版"纪录片"
- **AI 亮点：** AI 根据碎片组合实时生成叙事，不同排列组合产生截然不同的故事
- **开发要点：** 卡片选择界面 + AI 文本流式输出，纯前端方案

## Idea 6：《创世手记》— AI 世界构建

| 属性 | 评估 |
|------|------|
| **类型** | 沙盒模拟 + AI 生成 |
| **难度** | ★★★★☆（中高） |
| **CodeBuddy 适配度** | ★★★★★ — AI 世界演化逻辑、自然语言解析、状态管理，AI 高度契合 |
| **3人团队可行性** | ★★★☆☆ — 需要做世界状态的持久化和一致性管理，逻辑复杂度较高 |
| **独特性** | ★★★★★ — "用文字创造世界"的概念非常有黑客松精神，demo 效果炸裂 |

**玩法：**
- 你是一本"创世之书"的书写者，你写下的文字会变成真实世界
- 玩家输入自然语言描述（如"这个世界有漂浮的岛屿和会说话的动物"），AI 会生成这个世界的地理、种族、文明和冲突
- 玩家通过"书写"干预世界走向 — 写下一场战争、一个英雄、一场灾难，世界会随之演变
- **AI 亮点：** AI 负责世界演化的逻辑，玩家的创造力决定世界的样子
- **开发要点：** 文本输入 + AI 响应，核心是世界状态管理（防止 AI 生成矛盾）

## Idea 7：《双生侦探》— AI 对抗叙事

| 属性 | 评估 |
|------|------|
| **类型** | 推理解谜 + 双 AI 对抗 |
| **难度** | ★★★★☆（中高） |
| **CodeBuddy 适配度** | ★★★★★ — 双 AI 对话管理、矛盾检测、笔记系统，这是最能展现 AI 编程能力的项目类型 |
| **3人团队可行性** | ★★★★☆ — 纯文本+笔记系统，1人 Prompt 对抗设计，1人前端，1人逻辑整合 |
| **独特性** | ★★★★★ — 双 AI 各执一词的对抗设计是全新概念，评委演示效果极佳 |

**玩法：**
- 玩家扮演侦探审问两个 AI 嫌疑人，他们中一人说谎、一人说真话，玩家必须通过提问找出真相
- 两个 AI 各有一份"剧本" — 真话者知道完整真相，说谎者知道被篡改的版本。玩家自由提问（不限于选择题），AI 根据各自的剧本动态回答
- 侦探工具包：做笔记、对比供词矛盾点、心理施压
- **AI 亮点：** 两个 AI 各自持有不同的"事实版本"，对话逻辑是对抗式设计；说谎者的谎言水平随难度提升
- **开发要点：** 纯文本对话 + 笔记系统，核心是 AI prompt 对抗设计（如何让说谎 AI 编得可信又露出破绽）
