# 坐标体系规范

> **版本**: v1.0 | **创建日期**: 2026-05-25 | **适用范围**: 前后端全栈

---

## 一、核心原则

**后端存储/传输瓦片坐标（tile），前端负责瓦片 ↔ 像素转换。**

后端不感知地图渲染方式（代码生成 / 大图贴图 / Tiled Tileset），
只关心实体在地图网格中的离散位置。

---

## 二、坐标系定义

| 属性 | 值 |
|------|------|
| 原点 | 左上角 (0, 0) |
| 列方向 | X 轴，向右递增 |
| 行方向 | Y 轴，向下递增 |
| 地图尺寸 | 80 列 × 50 行（当前版本） |
| 单格像素 | 16 × 16 px（由前端 `GAME.TILE_SIZE` 控制） |

### 2.1 瓦片坐标（Tile Coordinate）

后端使用的标准坐标，格式：

```json
{ "col": 43, "row": 16 }
```

- `col`：列号，范围 `[0, MAP_COLS-1]`
- `row`：行号，范围 `[0, MAP_ROWS-1]`
- 值为整数，表示实体所在/目标所在的格子索引

### 2.2 像素坐标（Pixel Coordinate）

前端 Phaser 使用的坐标，格式：

```json
{ "x": 688, "y": 256 }
```

- 由瓦片坐标 × TILE_SIZE 换算而来
- `x = col × TILE_SIZE`
- `y = row × TILE_SIZE`
- 实体中心点通常在格子中心：`(x + TILE_SIZE/2, y + TILE_SIZE/2)`

## 三、前后端职责分离

```
┌─────────────────────────────────────────────────┐
│              坐标数据流                          │
│                                                 │
│  ┌──────────┐    初始位置     ┌──────────────┐   │
│  │ 前端设计  │ ──────────────→ │ 后端存储      │   │
│  │ 地图布局  │                │ (持久化)      │   │
│  └──────────┘                └──────┬───────┘   │
│       ↑                             │            │
│       │     返回位置数据              │            │
│       │     (API Response)           │            │
│       ├─────────────────────────────┘            │
│       │                                          │
│       │  COORD.toPixel(col, row) → x, y          │
│       │  COORD.toTile(px, py) → col, row         │
│       │                                          │
│  ┌────▼──────────────────────────────────────┐   │
│  │ 前端渲染：NPC 精灵 / 物品 / 任务目标标记    │   │
│  └───────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

### 3.1 后端职责

- 存储所有实体的 tile 坐标 `{col, row}`
- 在需要时更新实体位置（如剧情触发 NPC 移动）
- 通过 API 返回位置信息，**不进行像素转换**

### 3.2 前端职责

- 定义地图布局，确定 NPC / 物品 / 目标点的初始 tile 位置
- 调用 `COORD.toPixel()` 将 tile 坐标转换为像素坐标用于渲染
- 调用 `COORD.toTile()` 将像素坐标转换为 tile 坐标（碰撞/到达判定）
- NPC 移动完成后向后端上报新位置，保持一致性

## 四、API 字段规范

### 4.1 NPC 位置

```json
{
  "id": "npc_chen",
  "name": "陈师傅",
  "position": { "col": 43, "row": 16 },
  "scene": "teahouse"
}
```

- `position.col` / `position.row`：NPC 当前所在的 tile 坐标
- `scene`：语义标注（teahouse / stage / dock / temple），辅助 LLM 理解场景上下文

### 4.2 物品位置

```json
{
  "id": "item_urn",
  "name": "父亲的骨灰盒",
  "location": {
    "scene": "cemetery",
    "position": { "col": 35, "row": 8 }
  }
}
```

- `location.position`：可选，物品在地图上的固定位置（可拾取物品需要）
- 无 `position` 表示物品仅在背包中，无地图位置

### 4.3 子任务目标位置

```json
{
  "id": "st_001",
  "title": "找到老戏院",
  "mode": "explore",
  "target_position": { "col": 48, "row": 8 },
  "target_scene": "stage_ruin"
}
```

- `target_position`：可选，前端用于在地图上高亮目标区域
- 前端可根据此坐标显示箭头 / 虚线圈 / 小地图标记等引导 UI

### 4.4 NPC 移动指令（后端 → 前端）

通过 SSE done 事件下发：

```json
{
  "full_text": "...",
  "npc_move": {
    "npc_id": "npc_chen",
    "from": { "col": 43, "row": 16 },
    "to": { "col": 48, "row": 24 }
  }
}
```

- 前端收到后执行像素级 lerp 动画（`from_pixel` → `to_pixel`）
- 到达后上报后端

### 4.5 NPC 位置上报（前端 → 后端）

```json
POST /api/game/{session_id}/npc/position
{
  "npc_id": "npc_chen",
  "position": { "col": 48, "row": 24 }
}
```

响应：

```json
{
  "success": true,
  "npc_id": "npc_chen",
  "position": { "col": 48, "row": 24 }
}
```

## 五、前端转换工具

定义于 `frontend/src/config.js`：

```javascript
export const COORD = {
  /** 瓦片坐标 → 像素坐标（渲染用） */
  toPixel(col, row) {
    return {
      x: col * GAME.TILE_SIZE,
      y: row * GAME.TILE_SIZE,
    };
  },

  /** 像素坐标 → 瓦片坐标（碰撞/判定用） */
  toTile(pixelX, pixelY) {
    return {
      col: Math.floor(pixelX / GAME.TILE_SIZE),
      row: Math.floor(pixelY / GAME.TILE_SIZE),
    };
  },

  /** 瓦片中心像素坐标（实体放置用） */
  toPixelCenter(col, row) {
    const half = GAME.TILE_SIZE / 2;
    return {
      x: col * GAME.TILE_SIZE + half,
      y: row * GAME.TILE_SIZE + half,
    };
  },
};
```

## 六、地图替换兼容性

无论前端地图渲染方式如何切换，坐标转换只需修改 `COORD` 层：

| 渲染方式 | 影响 | 需要改的 |
|----------|------|---------|
| 程序化代码（当前） | 无影响 | 仅 `COORD` 中的 `TILE_SIZE` 引用 |
| 单张大图贴图 | 碰撞数据需重新对齐 | `mapData` 数组 + `COORD.toPixel()` |
| Tiled Tileset | 碰撞层由 Tiled 管理 | `COORD.toPixel()` 中的 tile 大小 |

**关键**：后端存储的 `{col, row}` 在任何渲染方式下都不需要改动。

## 八、注意事项

1. **坐标合法性**：后端收到前端上报的位置时应校验 `0 <= col < MAP_COLS` 且 `0 <= row < MAP_ROWS`
2. **碰撞区域**：后端不管理碰撞，碰撞完全由前端处理。后端只负责"实体应该在哪"
3. **地图版本**：如果未来地图尺寸变化，后端需要在 `GET /api/scripts` 响应中返回 `map_cols` / `map_rows` / `tile_size` 让前端感知
4. **向后兼容**：v1.0 后，所有 API 不再返回旧的 `{x, y}` 像素坐标字段
