## 阶段枚举

```python
class Stage(Enum):
    DISDAIN = 1    # 不屑 — 戏班众人冷眼相看
    UNDERSTAND = 2 # 了解 — 有人愿意说真心话
    CHOICE = 3     # 抉择 — 决定戏班的命运
```

## 阶段参数表（来自 `config.py` 的 `STAGES`）

| 阶段 | 名称 | 色调 | BGM | 对话基调 |
|------|------|------|-----|---------|
| 1 | 不屑 | `cold` | melancholy | 冷漠、疏离、话中带刺 |
| 2 | 了解 | `warm` | hopeful | 温和、敞开、偶有真情流露 |
| 3 | 抉择 | `dramatic` | intense | 情感浓烈、言辞恳切 |

## 阶段切换规则（来自 `config.py` 的 `STAGE_RULES`）

```python
STAGE_RULES = {
    # 1→2：任一 NPC 关系 >= 20，或触发关键事件 chen_tells_past，或对话轮数 >= 3

        "min_dialogue_rounds": 5,

```

## 阶段切换架构

### 加权判定引擎

```
每轮对话完成后（SSE done 事件生成时）：

   ┌──────────────────┐
   │  规则引擎 O(1)    │
   │                  │
   │ 条件1: any NPC   │
   │  relationship   │
   │  >= 阈值的规则    │
   │                  │
   │ 条件2: 关键事件   │
   │  已触发 → true   │
   └────────┬─────────┘
            │
     ┌──────┴───────┐
     │ 规则命中？     │
     └──┬──────┬────┘
        │      │
     Yes│      │No
        ▼      ▼
   立即切换  ┌──────────────────┐
            │ 独立 LLM 阶段判定  │
            │                  │
            │ Prompt（轻量）：   │
            │ "当前阶段{stage}， │
            │  关系值{rel}，    │
            │  已触发事件{evts}，│
            │  最近对话摘要。    │
            │  是否可推进？"     │
            │                  │
            │ 输出：            │
            │ {should_advance: │
            │  bool, reason}   │
            └────────┬─────────┘
                     │
              ┌──────┴───────┐
              │ LLM 判定推进？ │
              └──┬──────┬────┘
                 │      │
              Yes│      │No
                 ▼      ▼
        ┌────────────┐  跳过
        │ 单轮判定计数 │
        │             │
        │ 1 轮判定    │──── No ──→ 不切换，计数清零
        │ 即可推进？   │
        └──────┬──────┘
               │ Yes
               ▼
          执行阶段切换
```

| 判定来源 | 权重 | 切换条件 |
|---------|------|---------|
| 规则引擎 | **立即生效** | 任一条件达标 → 直接切换，无需 LLM 确认 |
| LLM 判定 | **单轮判断** | 单轮 LLM 独立判定为"可推进"即可切换 |
| 两者同时 | 规则优先 | 规则命中时不等 LLM 结果 |

### 切换伪代码

```python
async def advance_stage(session: GameSession, new_stage: Stage, reason: str):
    session.current_stage = new_stage
    for npc in session.npcs.values():
        npc.current_greeting = await generate_greeting(npc, new_stage)
    session.events_triggered.add(f"stage_{new_stage.value}_reached")
    session.stage_llm_consecutive = 0
    await db.update_session(session)

async def check_stage_advancement(session, events_this_turn):
    rules = STAGE_RULES.get(session.current_stage, {})

    # 规则引擎：以 config.py 中 STAGE_RULES 为准
    if any(npc.relationship >= rules.get("min_relationship", 50)
           for npc in session.npcs.values()):
        return await advance_stage(session, next_stage(session.current_stage), "关系值达标")

    key_events = rules.get("key_events", set())
    if any(e in key_events for e in session.events_triggered):
        return await advance_stage(session, next_stage(session.current_stage), "关键事件触发")

    # LLM 判定（单轮即可推进）
    result = await llm_client.stage_check(
        current_stage=session.current_stage,
        relationships={npc.id: npc.relationship for npc in session.npcs.values()},
        events=session.events_triggered,
        recent_summary=get_recent_summary(session),
    )
    if result.should_advance:
        session.stage_llm_consecutive += 1
        if session.stage_llm_consecutive >= 1:
            return await advance_stage(session, next_stage(session.current_stage), result.reason)
    else:
        session.stage_llm_consecutive = 0
    return False
```
