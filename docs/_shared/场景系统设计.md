# 场景系统设计

## 核心理念：世界先行，剧本后置

```
                          ┌──────────────────┐
                          │   开放世界资产     │
                          │                  │
     ┌──────────┐         │  ┌────────────┐  │
     │ 场景定义 │─────────│──│ scenes.yaml │  │
     │ (7个)    │         │  └─────┬──────┘  │
     └──────────┘         │        │         │
                          │  ┌─────▼──────┐  │
     ┌──────────┐         │  │ 物品清单    │  │
     │ 物品定义 │─────────│──│ items/      │  │
     │ (10件)   │         │  └─────┬──────┘  │
     └──────────┘         └────────┼─────────┘
                                  │
                          ┌───────▼────────┐
                          │  AI 剧本生成    │
                          │  ← 受世界约束   │
                          └───────┬────────┘
                                  │
                          ┌───────▼────────┐
                          │  微剧本骨架      │
                          └────────────────┘
```

- **场景属于世界**：场景是先于剧本存在的世界空间，不是为某个剧本服务的
- **物品属于场景**：物品归属于某个场景坐标，物品 + 场景 = 完整的物理空间描述
- **AI 生成剧本受世界约束**：AI 必须在已有场景和物品的框架内创作故事

## 数据层

### 文件：`data/world/scenes.yaml`

世界场景包含 7 个场景：

| scene_id | 名称 | 类型 | 子场景 | 氛围 |
|---|---|---|---|---|
| town | 江南小镇 | outdoor | - | 水乡、日常、安宁之下暗流涌动 |
| stage_ruin | 破旧戏台 | indoor | stage | 破败、寂静、檀香与霉味 |
| teahouse | 如意茶馆 | indoor | tea_house | 喧闹、茶香、消息集散地 |
| temple | 柳家祠堂 | indoor | ancestral_hall | 肃穆、昏暗、香火、遗忘与不甘 |
| father_house | 父亲旧居 | indoor | fathers_house | 尘封、孤独、旧日痕迹 |
| cemetery | 镇外墓地 | outdoor | graveyard | 荒凉、悲伤、未说完的话 |
| dock | 渡口码头 | outdoor | dock | 水边、黄昏、等待、离别的预感 |

### 字段说明

| 字段 | 含义 | 示例 |
|---|---|---|
| scene_id | 场景唯一ID（后端数据标签） | father_house |
| name | 中文名称 | 父亲旧居 |
| type | outdoor / indoor | indoor |
| description | 场景氛围描述（50-100字） | 镇南窄巷尽头的老宅... |
| atmosphere | 氛围关键词列表 | [尘封, 孤独, 旧日痕迹] |
| subscene_id | 前端子场景ID（null=主世界） | fathers_house |
| map_image | 地图图片文件名 | fathers_house.png |
| dimensions | 像素尺寸 | {w: 1387, h: 1134} |
| connections | 可通往的场景列表 | [town] |
| notable_areas | 场景内值得注意的区域 | [堂屋, 里屋, 照片墙] |
| default_npcs | 场景预设可能的NPC类型 | [卖菜大婶, 河边洗衣妇] |

## 场景连接拓扑

```
                    ┌─────────┐
          ┌─────────│  小镇   │──────────┐
          │         │ (town)  │          │
          │         └────┬────┘          │
          │              │               │
    ┌─────▼─────┐  ┌────▼─────┐  ┌──────▼──────┐
    │  破旧戏台  │  │  茶馆    │  │  渡口码头    │
    │stage_ruin │  │teahouse  │  │   dock      │
    └───────────┘  └──────────┘  └─────────────┘
          │              │               │
          └──────────────┼───────────────┘
                    ┌────▼────┐
                    │  祠堂   │
                    │ temple  │
                    └─────────┘
                    ┌────▼──────┐
                    │ 父亲旧居   │
                    │father_house│
                    └───────────┘
                    ┌────▼──────┐
                    │ 镇外墓地   │
                    │ cemetery  │
                    └───────────┘
```

所有子场景连接回镇中心（town），形成星形拓扑。玩家始终从镇的某个位置出发，进入子场景后返回镇中心。

## API

### `GET /api/world/scenes`

返回所有世界场景定义。返回格式：

```json
{
  "total": 7,
  "outdoor_count": 3,
  "indoor_count": 4,
  "adjacencies": {
    "town": ["stage_ruin", "teahouse", "dock", "temple", "father_house", "cemetery"],
    "stage_ruin": ["town"],
    "teahouse": ["town"],
    ...
  },
  "scenes": [
    {
      "scene_id": "stage_ruin",
      "name": "破旧戏台",
      "type": "indoor",
      "description": "一座废弃已久的戏台...",
      "atmosphere": ["破败", "寂静", "檀香与霉味", "辉煌不再"],
      "connections": ["town"],
      "notable_areas": ["前台戏台", "后台化妆间", "角落的旧木箱", "墙上挂着的京胡"]
    }
  ]
}
```

### `GET /api/world/items`

返回所有世界物品（按场景分组可发现物品）。详见 [物品系统设计](./物品系统设计.md)。

## AI 剧本生成集成

`POST /api/scripts/generate` 在生成 prompt 时自动注入场景信息和物品信息：

```
## 开放世界场景（重要）
以下是当前开放世界中已存在的所有场景。
你的剧本必须基于这些已有场景来设计。

### 江南小镇（town）[户外]
氛围: 小镇、水乡、日常、安宁之下暗流涌动
描述: 一座被水网环绕的江南小镇...
可前往: stage_ruin | teahouse | dock | temple | father_house | cemetery

### 破旧戏台（stage_ruin）[室内]
氛围: 破败、寂静、檀香与霉味、辉煌不再
描述: 一座废弃已久的戏台...
重要区域: 前台戏台、后台化妆间、角落的旧木箱、墙上挂着的京胡

...

## 开放世界预置物品（重要）
以下是当前开放世界中已存在的所有预置物品...
（物品列表，按场景分组）
```

AI 同时获得场景和物品两个维度的约束，生成的剧本必须：
1. 故事发生在已有场景中（不能说"主角走进一个不存在的地下室"）
2. 叙事道具必须是已有物品（不能说"发现了一本不存在的日记"）
3. 人物动线必须遵循场景连接拓扑（不能从戏台直接跳到墓地而不经过小镇）

## 如何扩展场景

### 新增一个子场景

1. **准备地图图片**：放在 `frontend/public/assets/images/maps/`
2. **注册到前端**：在 `frontend/src/scenes/modules/SubSceneConfig.js` 中添加条目
3. **注册到世界数据**：在 `data/world/scenes.yaml` 中添加场景定义
4. **设置连接**：在 `connections` 字段中指定从哪个主世界位置可进入
5. **预设物品**：在 `data/world/items/world_items.yaml` 中添加属于该场景的物品

所有修改完成后，AI 下次生成剧本时会自动包含新场景的信息。

### 修改已有场景描述

只需编辑 `data/world/scenes.yaml` 中对应场景的 `description`、`atmosphere`、`notable_areas` 字段。无需改代码——API 返回的是动态读取的 YAML 数据。

### 场景与物品的同步

场景和物品的关联通过 `scene` 字段自动维护。`GET /api/world/items` 返回的每个物品都带 `scene` 属性，前端/后端据此判断物品应出现在哪个子场景中。

## 设计原则

### 世界层 vs 剧本层

| 层级 | 职责 | 文件 |
|---|---|---|
| **世界层** | 场景的物理存在（空间、氛围、连接关系） | `data/world/scenes.yaml` |
| **世界层** | 物品的物理存在（名称、描述、位置） | `data/world/items/` |
| **剧本层** | 物品的叙事角色（关键/普通、章节关联、NPC关联） | `data/scripts/{id}/items/` |
| **运行时层** | 玩家即时状态（NPC位置、碰撞、发现标记） | localStorage / session |

### 单向依赖

世界层 → 不依赖任何剧本或运行时状态
剧本层 → 可引用世界资产（通过 scene 标签和 item_id）
运行时层 → 从世界层和剧本层加载数据

### YAML 作为唯一数据源

场景和物品的定义都是 YAML 文件，不是代码中的硬编码。修改世界数据不需要改代码，只需改 YAML 后重启（或有 reload 机制）即可生效。
