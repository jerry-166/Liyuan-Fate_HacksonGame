# 物品系统设计

## 核心理念：世界先行，剧本后置

```
┌──────────────────────────────────────────────┐
│                 开放世界                       │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐  │
│  │ 场景信息  │  │ 世界物品  │  │  NPC 角色  │  │
│  │ (scenes) │  │ (items)  │  │  (npcs)   │  │
│  └────┬─────┘  └────┬─────┘  └─────┬──────┘  │
│       │              │              │         │
│       └──────────────┼──────────────┘         │
│                      │                        │
│               ┌──────▼───────┐                 │
│               │   AI 剧本生成  │  ← 受世界约束   │
│               └──────┬───────┘                 │
│                      │                        │
│               ┌──────▼───────┐                 │
│               │   微剧本骨架   │                │
│               └──────────────┘                │
└──────────────────────────────────────────────┘
```

- **世界是固定的**：场景、物品、NPC 是先于剧本存在的世界资产
- **剧本是 AI 基于世界生成的**：AI 必须利用已有世界资产来编故事，不能凭空创造不存在的物品或场景
- **物品属于世界**：每件物品只归属于一个场景坐标，不绑定任何剧本、章节、NPC

## 数据层

### 目录结构

```
data/world/items/
├── templates.yaml      ← 通用物品模板（物理属性）
└── world_items.yaml    ← 世界物品清单（具体实例）
```

### 通用物品模板 (`templates.yaml`)

模板描述物品的**客观物理属性**，不含叙事色彩，跨剧本可复用。

当前有 8 个模板：

| template_id | category | name | size | holdable | actions |
|---|---|---|---|---|---|
| tmpl_child_costume | clothing | 戏服 | medium | true | examine, show |
| tmpl_script_book | document | 戏本 | small | true | examine, read, show |
| tmpl_old_photo | misc | 老照片 | tiny | true | examine, show |
| tmpl_trunk | container | 旧木箱 | large | false | examine, open |
| tmpl_jinghu | instrument | 京胡 | medium | true | examine, hold |
| tmpl_urn | misc | 骨灰盒 | small | true | examine |
| tmpl_genealogy | document | 族谱 | small | true | examine, read |
| tmpl_tablet | misc | 牌位 | tiny | false | examine |

每个模板包含字段：
- `template_id` — 模板唯一标识，世界物品通过此字段引用
- `category` — 物品类别（clothing / document / misc / container / instrument）
- `name` — 通用名称
- `generic_desc` — 通用物理描述（AI 可据此生成具体物品的外观）
- `size` — 体积（tiny / small / medium / large）
- `icon` — 前端图标标识
- `holdable` — 是否可拾取/持有
- `actions` — 可执行的交互动作

### 世界物品 (`world_items.yaml`)

世界物品引用模板获得物理属性，叠加自身的叙事描述和位置信息。

当前有 10 件世界物品：

| item_id | 名称 | 模板 | 场景 | 坐标 |
|---|---|---|---|---|

| item_father_script | 父亲的手抄剧本 | tmpl_script_book | father_house | (20, 5) |
| item_childhood_photo | 童年合影 | tmpl_old_photo | father_house | (24, 5) |
| item_father_single_photo | 父亲旧照（单人） | tmpl_old_photo | father_house | (23, 4) |
| item_genealogy | 柳家族谱 | tmpl_genealogy | temple | (34, 12) |
| item_old_trunk | 旧戏服木箱 | tmpl_trunk | stage_ruin | (14, 10) |
| item_old_jinghu | 墙上的老京胡 | tmpl_jinghu | stage_ruin | (12, 8) |
| item_stage_group_photo | 戏班合影 | tmpl_old_photo | teahouse | (34, 16) |
| item_temple_tablet | 柳家祖先牌位 | tmpl_tablet | temple | (36, 8) |

世界物品**只包含**这些字段：
- `item_id` — 物品唯一标识
- `name` — 物品名称
- `template_ref` — 引用的模板 ID
- `base_description` — 物品描述（AI 生成或人工预设）
- `scene` — 所在场景
- `position` — 坐标 `{col, row}`（80x50 tile 地图）
- `desc_source` — 描述来源（`fixed` 预设 / `ai` AI 生成）

**不包含**：章节关联、NPC 关联、关键道具标记、获取方式——这些是剧本层面的属性，不属于世界层。

## API 接口

### `GET /api/world/items` — 世界物品全量查询

返回开放世界中所有预置物品的完整信息（模板合并后）。**无章节、无 NPC、无关键标记**——纯粹的世界资产数据。这是 AI 生成微剧本的核心数据源。

**响应格式**：

```json
{
  "total": 10,
  "templates_count": 8,
  "scenes": ["cemetery", "father_house", "stage_ruin", "teahouse", "temple"],
  "items": [
    {
      "item_id": "item_child_costume",
      "name": "孩童戏服",
      "template_ref": "tmpl_child_costume",
      "base_description": "一件小号的戏曲戏服，红底金线绣传统纹样...",
      "generic_desc": "一件小号的戏曲戏服，约5-8岁孩童尺寸...",
      "desc_source": "fixed",
      "category": "clothing",
      "icon": "costume",
      "size": "medium",
      "holdable": true,
      "actions": ["examine", "show"],
      "scene": "father_house",
      "position": {"col": 23, "row": 7}
    }
  ]
}
```

**合并逻辑**：模板提供物理属性（category / icon / size / holdable / actions），世界物品可覆盖这些字段。名称和描述优先使用世界物品的定义，fallback 到模板的通用值。

## 数据流

### 物品定义流程

```
人工 / AI 预设
      │
      ▼
┌─────────────────────────┐
│  data/world/items/       │
│  ├── templates.yaml      │  ← 模板（物理属性）
│  └── world_items.yaml    │  ← 物品实例（名称 + 描述 + 位置）
└───────────┬─────────────┘
            │ 读取
            ▼
┌─────────────────────────┐
│  routes/world.py        │
│  GET /api/world/items   │  ← 合并模板+物品，输出干净数据
└───────────┬─────────────┘
            │ 数据源
            ▼
┌─────────────────────────┐
│  routes/script.py        │
│  POST /api/scripts/      │
│  generate                │  ← AI 生成剧本时注入物品列表
│                          │     作为「世界约束」
└─────────────────────────┘
```

### AI 剧本生成中的物品注入

`POST /api/scripts/generate` 在生成剧本前，自动加载世界物品列表并注入到 prompt 中，格式如下：

```
## 开放世界场景与物品（重要）
以下是当前开放世界中已存在的场景和预置物品。
你的剧本必须基于这些已有的物品来设计——
不要凭空编造不存在的物品或场景。
但你可以：让物品之间产生隐藏的联系、
赋予物品新的叙事意义、
设计 NPC 对物品的不同认知。
——总之：用已有的东西，讲新的故事。

### 父亲旧居（father_house）
- [clothing] 孩童戏服 (23,7) (可拾取)
  一件小号的戏曲戏服，红底金线绣传统纹样...
- [document] 父亲的手抄剧本 (20,5) (可拾取)
  一本泛黄的手抄戏曲台本...

### 戏台遗址（stage_ruin）
- [container] 旧戏服木箱 (14,10) (仅查看)
  一口老旧的大木箱...

...
```

## 设计原则

### 世界层 vs 剧本层

| 层级 | 职责 | 包含字段 | 文件位置 |
|---|---|---|---|
| **世界层** | 物品的物理存在 | item_id, name, template_ref, base_description, scene, position, desc_source | data/world/items/ |
| **剧本层** | 物品的叙事角色 | stage_relevance, related_npcs, is_key, acquire_method, npc_knowledge | data/scripts/{id}/items/ |

剧本可以给世界物品叠加叙事属性（例如：在某剧本中「孩童戏服」是关键证据），但物品本身的存在不依赖于任何剧本——即使没有剧本，这件戏服也在父亲旧居的 (23,7) 位置。

### 模板复用

模板是跨剧本复用的基础能力。新增物品时，如果现有模板能描述其物理属性，直接引用即可；如果新物品的物理属性与现有模板差异较大，创建新模板。

### 坐标系统

- 地图尺寸：80 x 50 tiles
- 坐标格式：`{col, row}`，col 为列（x 轴），row 为行（y 轴）
- 每个物品有唯一的 scene + position 组合

## 扩展方向

### 剧本工坊集成（已实现）

世界物品浏览功能已集成到剧本工坊（`editor.html`）中。工坊侧边栏新增「世界资产」→ 📦 世界物品视图：

- 调用 `GET /api/world/items` 获取全量物品数据
- 自适应卡片网格展示（物品图标/名称/分类/尺寸/所在场景/描述）
- 支持按分类筛选（衣物/文书/杂项/容器/乐器）
- 数据来源与 `data/world/items/` YAML 文件一致，确保「世界先行，剧本后置」

### 未来场景信息接入

计划将场景信息也纳入世界数据层（`data/world/scenes/`），在 AI 生成剧本时同样注入为「世界约束」，让 AI 生成的叙事与场景空间结构一致。

```
data/world/
├── items/
│   ├── templates.yaml
│   └── world_items.yaml
└── scenes/                    ← 已实现（scenes.yaml）
    └── scenes.yaml
```

### 物品的 AI 描述生成

当前物品描述（`base_description`）为人工预设。未来可对接多模态生成能力，根据模板的 `generic_desc` + 场景氛围自动生成更丰富的物品外观描述。
