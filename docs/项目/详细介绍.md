# 梨园生死 — 项目详细介绍

## 一、项目概述

**梨园生死**是一款由 AI Agent 驱动的 2D 开放探索叙事游戏。玩家扮演归乡的戏班继承人，在江南小镇中自由探索、与 AI NPC 深度对话、收集关键物品、推进六章剧情，最终收获专属结局。

## 二、核心理念：AI 驱动，而非预设叙事

### 传统叙事游戏的局限

传统叙事游戏的所有剧情、对话、分支都是**固定预设**的——玩家只是在已有路径中做选择，每一个分支、每一种结局、每一句对话在开发阶段就已经写好，无法让不同玩家感受到真正差异化的体验。

### 我们的解法

- **NPC 有自己的人格**：每个 NPC 拥有独立的人设、性格、爱好，AI 根据关系值动态调整其行为与语言
- **剧情不做过多约束**：全局由 AI 驱动，开发者只提供微剧本骨架，AI 负责"生长出血肉"
- **同一 NPC，不同玩家的体验完全不同**：你对他好，他倾囊相授；你惹怒他，他拒绝相助

## 三、核心玩法

### 3.1 章节渐进式生成

游戏启动时，AI 根据微剧本（序章 + 各章节子标题）生成五章内容大纲。每完成一章，AI 会对该章所有 NPC 对话进行总结、摘取关键事件，再次调用总控 AI 规划下一章的详细剧情。**一章章推进，剧本不断完善。**

> "开发者提供框架，AI 生长出血肉。"

### 3.2 AI 实时 NPC 对话

五个核心 NPC 由 AI 实时扮演：
- 根据当前章节、历史事件、与玩家的关系值动态生成回复
- 支持流式输出，对话自然流畅
- 关系值变化 → 行为语言随之改变

### 3.3 场景探索与物品交互

地图上各子场景散布可交互物品：
- 通用物品 + 特殊剧情物品
- 拾取物品后可与 NPC 交互，触发不同事件对话
- 推进剧情发展，解锁新的分支走向

### 3.4 章节间总结与规划

每章结束时执行：
1. 总结该章所有 NPC 对话
2. 摘取关键事件
3. 调用总控 AI 规划下一章详细剧情
4. 前章所作所为直接影响后续走向，形成真实因果链

### 3.5 专属结局生成

通关时的结局不是 A/B 预设选择，而是：
- 回顾整局所有对话、选择、行为
- 汇总与各 NPC 的关系变化
- 生成**专属于你的结局**：包含关键事件回顾 + 各 NPC 与你未来的发展走向

## 四、核心优势：可拓展性

### 4.1 预设大世界 + AI 筛选

我们的设计理念是**先创造一个完整的大世界**：
- NPC 已预设好（个性、爱好、背景故事）
- 物品已预设好（通用物品 + 特殊物品体系）
- 场景已预设好（多个子场景 + 主场景）

AI 的任务是：**从大世界中选择剧本需要的 NPC、物品、子场景，填充完善剧本内容。**

### 4.2 剧本可插拔

同一套框架，更换微剧本文件即可演绎完全不同的故事：
- 江南小镇 → 赛博朋克
- 古代武侠 → 现代都市
- 底层架构无需任何改动

用户只需提供序章 + 三四个子标题，AI 即可驱动完整剧情。

### 4.3 物品体系设计

- **通用物品**：跨剧本复用
- **特殊物品**：特定剧本专用
- 独特剧本如需独特物品，AI 可灵活适配

## 五、未来愿景

| 方向 | 说明 |
|------|------|
| **场景世界观拓展** | 持续添加场景、NPC、物品，AI 可筛选范围更大，剧本可选性更强 |
| **剧本可插拔** | 不同微剧本演绎不同故事，底层架构无需改动 |
| **AI 辅助微剧本生成** | 玩家与 AI 交流想法，AI 辅助完成微剧本设定，降低创作门槛 |
| **人机共创** | AI 生成 MVP 版本，玩家在此基础上添加想法、指定 NPC/物品/场景，进一步完善剧本 |
| **NPC 关系网** | 从"玩家-NPC"关系扩展为"NPC-NPC"关系网络，AI 感知关系张力，让世界自己运转 |
| **创意工坊** | 玩家上传和分享自创微剧本，从单机游戏进化为叙事内容平台 |

## 六、总结

梨园生死不仅是一款游戏，更是一套 **AI 叙事游戏框架**。

**叙事的上限不再是开发者的写作上限，而是 AI 与玩家共同创造的上限。**

每一次游玩都是独一无二的故事，每一个结局都只属于你。
