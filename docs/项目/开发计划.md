# 开发计划

> 从架构设计文档中提取的开发计划和后续版本规划。
>
> **当前状态（2026-05-31）**：Phase 1-6 全部完成。v1.0 完整可玩版已交付，包含 6 章剧情、5 个 AI NPC、子场景系统、物品系统、存档系统、BGM、任务面板、结局生成等全部核心功能。

## 一、MVP 开发优先级

| 优先级 | 模块 | 预估工时 | 依赖 | 说明 |
|--------|------|---------|------|------|
| **P0** | `main.py` + `config.py` | 0.5h | 无 | FastAPI 骨架 + 配置加载 |
| **P0** | `state/session.py` + `state/manager.py` | 1h | 无 | 会话状态是一切的基础 |
| **P0** | `llm/client.py` | 1h | config | LLM 流式调用封装 |
| **P0** | `routes/dialogue.py` (SSE) | 2h | state + llm | **核心链路** |
| **P1** | `agents/prompt_builder.py` | 1.5h | state | Prompt 模板 + 人设注入 |
| **P1** | `agents/npc_agent.py` | 1h | prompt_builder + llm | NPC Agent 协调层 |
| **P1** | `agents/personas/*.yaml` | 1h | 需 C 提供内容 | 两个 NPC 人设卡 |
| **P1** | `routes/game.py` (start + state) | 1.5h | state + manager + personas | 游戏开始 + 状态查询 |
| **P2** | `state/stage_engine.py` | 1h | state | 双模阶段切换判定 |
| **P2** | `storage/database.py` + `schema.sql` | 2h | state | SQLite 持久化 |
| **P3** | `agents/response_parser.py` | 1.5h | llm | LLM 响应结构化提取 |
| **P3** | `routes/game.py` (evaluate) | 1h | state + llm | 结局评价生成 |
| **P3** | `state/summarizer.py` (占位) | 0.5h | 无 | 接口预留，MVP 后启用 |

**总预估**: P0~P1 约 9.5h，完整 MVP 约 14.5h。

## 二、开发顺序

```
Phase 1: 骨架搭建 (P0)
  main.py → config.py → state 数据类 → state manager
  → 验证：能创建/读取 GameSession

Phase 2: LLM 打通 (P0)
  llm/client.py → 简单的 test stream
  → 验证：能调通腾讯云 LLM，逐 token 打印

Phase 3: 核心对话链路 (P0-P1)
  routes/dialogue.py → prompt_builder → npc_agent
  → 验证：前端 F 键 → SSE 流式显示 NPC 回复

Phase 4: NPC 个性 + Handoff (P1)
  personas/*.yaml → prompt 注入人设 + 事件
  → 验证：两个 NPC 语气不同，cross-NPC 事件生效

Phase 5: 阶段切换 (P2)
  stage_engine.py → 规则判定 → LLM 双模判定
  → 验证：对话 N 轮后阶段切换，前端色调变化

Phase 6: 持久化 + 评价 (P2-P3)
  SQLite → 对话存储 → evaluate 接口
  → 验证：刷新页面不丢数据，结局评价正常生成
```

## 三、后续版本规划（非 MVP）

| 功能 | 说明 | 预估版本 |
|------|------|---------|
| NPC 主动对话 | WebSocket + 行为树，NPC 可主动找玩家说话 | v2.0 |
| NPC 工具调用 | NPC Agent 可调用"查询状态""搜索记忆"等工具 | v2.0 |
| NPC 自主移动 | 后端路径规划 → 前端同步位置 | v2.0 |
| 对话摘要 | 上下文窗口管理，长对话自动压缩 | v1.5 |
| 自由文字输入 | 玩家可打字输入，而非仅点选选项 | v1.5 |
| 多结局分支 | 4+ 种结局，依关系值和关键选择判定 | v2.0 |
| 用户系统 | 注册/登录，多存档 | v3.0 |
| WebSocket 双向通信 | 支持 NPC 主动推送 + 实时状态同步 | v2.0 |
