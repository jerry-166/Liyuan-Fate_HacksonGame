## 二、架构模式

### Phaser Scene 场景驱动架构

| 层级 | 对应概念 | 说明 |
|------|---------|------|
| **Scene 层** | 类似"视图层" | 每个 Scene 管理一组完整的游戏画面和逻辑 |
| **Event Emitter 层** | 类似"消息总线" | Scene 间通过 `scene.events` 进行解耦通信 |
| **DOM 辅助层** | 原生输入覆盖 | `index.html` 中 `<input>` 等 DOM 元素，由 Scene 控制显隐，用于自由文本输入 |
| **API 层** | 数据访问层 | `src/api/client.js` 封装所有后端通信（16 个 API 函数 + 4 个本地存档槽位函数，Mock + 真实双模式） |
| **Config 层** | 常量配置 | `src/config.js` 存放游戏常量、配色、场景 key |

### 目录结构

```
frontend/
├── index.html             # 入口 HTML（含 DOM 层自由输入组件）
├── CHANGELOG-2026-05-24.md # 本次会话变更总结
├── package.json           # 依赖：仅 phaser + vite
├── vite.config.js         # Vite 配置（proxy /api 到 8000）
└── src/
    ├── main.js            # 游戏启动入口，Phaser.Game 实例化
    ├── config.js          # 全局常量（尺寸、颜色、场景 key、阶段色调）
    ├── api/
    │   └── client.js      # API 客户端（12 个函数，Mock + 真实双模式）
    └── scenes/
        ├── BootScene.js   # 启动场景（空加载，直接转 GameScene）
        ├── MenuScene.js   # 主菜单 + 存档管理面板（列表、滚动、继续/删除）
        ├── GameScene.js   # 核心游戏场景（地图、玩家、NPC、交互、阶段色调）
        └── UIScene.js     # UI 覆盖层（对话UI、SSE流式、选项、自由输入、历史面板、结局）
```

## 三、Scene 分析

### 3.1 main.js — 游戏入口

```javascript
const config = {
  type: Phaser.AUTO,
  parent: 'game-container',
  width: 1024, height: 768,
  backgroundColor: '#1a1a2e',
  pixelArt: true,
  roundPixels: true,
  dom: { createContainer: true },              // ← 启用 DOM 元素层
  scale: { mode: Phaser.Scale.FIT, autoCenter: Phaser.Scale.CENTER_BOTH },
  physics: { default: 'arcade', arcade: { gravity: { x: 0, y: 0 }, debug: false } },
  scene: [BootScene, MenuScene, GameScene, UIScene],
};
```

同时导出全局自由文本输入工厂函数 `createGlobalInput()` 和 `globalInputValues`，供 UIScene 在对话中显示 DOM `<input>` 元素实现自由文本输入。

### 3.2 BootScene — 启动场景

最小化启动器，`preload()` 为空，`create()` 直接转入 MenuScene。

### 3.3 MenuScene — 主菜单 + 存档管理

提供"开始游戏"和"继续游戏"两个核心按钮。点击"继续游戏"直接弹出存档列表面板（`showArchivePanel()`），调用 `GET /api/sessions` 从后端获取所有会话，渲染带"继续"/"删除"按钮的卡片式列表。面板支持滚轮滚动、ESC 关闭、遮罩点击关闭。

### 3.4 GameScene — 核心游戏场景（640 行）

#### 地图系统
- **程序化生成**（非 Tiled tilemap）：80×50 格子，每格 16px = 1280×800 世界
- 12 种 Tile 类型（草地、青石板路、土路、河水、石桥、戏台、茶馆、码头、祠堂、民居、树木）
- 每种 Tile 有 3 色调色板，配合棋盘格实现视觉变化
- 程序化布局：主街、小河、桥、戏台区域、茶馆、码头、祠堂、散落民居

#### 碰撞系统
所有建筑物和自然障碍不可穿越（`GRASS` / `ROAD` / `DIRT` / `BRIDGE` 可通过）。子系统区域入口（茶馆内部等）尚无切换逻辑。

#### 玩家系统
- 程序化绘制 14×28 像素 Q 版角色
- WASD + 方向键双模式控制，斜向速度归一化
- `setCollideWorldBounds(true)` + Arcade Physics collider

#### NPC 系统
5 个 NPC（从后端 API 状态动态创建）：陈师傅（茶褐色长衫、白发）、小华（蓝色短衫、黑发）、老周（灰褐色长衫、银灰发）、梅姨（暖褐色衣衫、深棕发）、船夫老李（棕灰色短衫、灰发）。NPC 通过 `refreshNPCsFromState()` 从后端 API 返回的 `npcs` 数组动态生成，支持运行时新增/隐藏/更新。头顶气泡显示 `current_greeting` 问候语，物理 `overlap` 检测玩家接近（64px 范围），接近时显示"按 [F] 与 XX 对话"。

#### 交互流程
```
player overlap npc → 显示 [F] 提示 → JustDown(F)
→ events.emit('dialogue:start', {npcId, name})
→ UIScene 监听并弹出对话 UI
```

### 3.5 UIScene — UI 覆盖层（约 1250 行，核心场景）

UIScene 运行于 GameScene 之上（`this.scene.launch('UIScene')`），通过 `events.on()` 监听 GameScene 事件驱动全部 UI 逻辑。

#### 对话系统
- **对话面板**（`createDialogPanel`）：底部 210px 高的半透明面板，含 NPC 名字、文本滚动区域（带 GeometryMask 裁剪 + 滚动条）、选项按钮区
- **SSE 流式显示**：`parseSSEStream` 回调驱动，`onDelta` 逐 chunk 追加文本 + 自动滚底，`onDone` 触发分页
- **文本分页**（`splitTextToPages`）：二分查找截断点，超长文本自动分多页显示，支持点击翻页（右下角页码提示）
- **选项按钮**：动态宽度计算，二次校验不超面板边界，鼠标 hover 高亮，数字键 1-3 快捷选择
- **自由文本输入**：对话时底部显示 DOM `<input>`（通过 `createGlobalInput()` 工厂），支持自由输入文字按 Enter 发送，与预设选项共用 `onOptionSelected` 逻辑调用 SSE 流式接口
- **物品展示对话**（v2 新增）：通过 `showItemToNpcStream` 向 NPC 展示背包物品，触发特殊对话

#### 章节系统集成（v2 新增）
- **章节开始**：GameScene 初始化时调用 `startChapter()` 获取当前章节和子任务
- **阶段过渡动画**（`playStageTransition`）同步使用 `STAGE_TONES` 中的章节信息（名称、色彩、描述）
- **子任务追踪**：`getTask()` 获取各 NPC 对话完成度（`npc_completion_votes`）驱动章节推进

#### 记忆回想（历史面板）
- **全屏历史面板**（`createHistoryPanel`）：按 H 键切换，GeometryMask 裁剪内容区，滚轮滚动
- **数据源**：`refreshHistoryContent()` 异步调用 `GET /api/game/{id}/dialogues` 获取该 session 完整对话历史（含所有轮次），失败回退内存数据
- **智能标签**：`lastSpeaker` 追踪说话人变化，同一 NPC 连续发言不重复显示名字标签

#### 存档槽位系统（v2 新增）
- **快捷存档**：游戏中按数字键 1-6 调用 `saveToSlot()` 存入 localStorage 槽位
- **快捷读档**：按 Ctrl+数字键 1-6 调用 `loadFromSlot()` 从槽位恢复
- **槽位显示**：存档面板显示 6 个槽位，含阶段名和时间标签

#### 阶段过渡与结局
- **阶段过渡动画**（`playStageTransition`）：全屏深色遮罩 + 章节标题 + 描述淡入，停留 2.2s 后淡出
- **结局画面**（`triggerEndingSequence`）："命运的齿轮开始转动……" → 调用 `POST /api/game/{id}/evaluate` → 逐行显示标题/关键节点/人生感悟/NPC 结局

#### 输入与状态管理
- **ESC 关闭**：对话框翻页中 ESC 跳过翻页，选项时 ESC 关闭对话（调用 `exitDialogue` API 非阻塞获取告别语）
- **自由输入保护**：DOM 输入框聚焦时跳过 Phaser 键盘事件处理
- **错误处理**：`buildErrorMessage()` 按错误码（SESSION_NOT_FOUND / NPC_NOT_FOUND / NETWORK_ERROR）返回中文提示

## 四、数据流向

```
[用户操作]
    │
GameScene.update()              ← Phaser 游戏循环（60fps）
    │
    ├── WASD → player.setVelocity()
    ├── overlap 检测 → 显示 [F] 提示
    └── JustDown(F) → emit('dialogue:start')
                           │
                     UIScene (监听事件)
                           │
              ┌────────────┼────────────┐
              │            │            │
      预设选项按钮    自由文本输入    H键历史面板
      (数字键+点击)  (Enter发送)   (全屏记忆回想)
              │            │            │
              └────────────┼────────────┘
                           │
                     api/client.js
           ┌───────────────┼───────────────┐
           │               │               │
    startDialogueStream  getDialogues   exitDialogue
    (POST /api/dialogue  (GET /api/game/ (POST /api/dialogue
     → SSE 流式)          {id}/dialogues) /exit)
    showItemToNpcStream  getItems        evaluateEnding
    (POST /api/dialogue  (GET /api/game/ (POST /api/game/
     /show-item → SSE)   {id}/items)     {id}/evaluate)
           │
    parseSSEStream
    ├── onDelta(chunk) → dialogText 逐字追加
    ├── onDone(result) → 分页 + 选项 + 章节切换/结局
    └── onError(err)   → buildErrorMessage()

[章节流程 — v2 新增]
GameScene.initGame() → startChapter()
    │
    └── 返回 task (sub_tasks + npc_completion_votes)
         → UIScene 根据 task 更新 UI
         → 各 NPC 对话完成后推进章节

[存档管理]
MenuScene.onContinue() → showArchivePanel()
    │
    ├── getSessions()     → GET /api/sessions → 渲染存档卡片列表
    ├── deleteSession(id) → DELETE /api/game/{id} → 删除 + 刷新列表
    └── loadArchive(id)   → scene.start('GameScene', {savedSessionId: id})
                              → GameScene.restoreGame(id)
                                  ├── getGameState(id) → GET /api/game/{id}
                                  ├── startChapter(id) → 恢复章节上下文
                                  └── 失败回退 localStorage

[本地存档槽位 — v2 新增]
游戏中按数字键 1-6
    ├── saveToSlot(sessionId, gameState, slotId) → localStorage 6 槽位
    └── 按 Ctrl+数字键 → loadFromSlot(slotId) → 恢复游戏
```

## 五、API 客户端

> 📎 **详见 [后端/API设计文档.md](../后端/API设计文档.md)** — 完整的接口请求/响应格式。

`api/client.js`（v2 章节驱动版）暴露以下函数：

| 函数 | HTTP | 路由 | 状态 |
|------|------|------|------|
| `startGame(playerName, scriptId)` | POST | `/api/game/start` | ✅ |
| `getGameState(sessionId)` | GET | `/api/game/{id}` | ✅ |
| `getScripts()` | GET | `/api/scripts` | ✅ v2 |
| `startChapter(sessionId, chapterId?)` | POST | `/api/game/{id}/chapter/start` | ✅ v2 |
| `getChapter(sessionId)` | GET | `/api/game/{id}/chapter` | ✅ v2 |
| `getTask(sessionId)` | GET | `/api/game/{id}/task` | ✅ v2 |
| `startDialogueStream(sessionId, npcId, playerMessage?)` | POST | `/api/dialogue`（SSE） | ✅ |
| `showItemToNpcStream(sessionId, npcId, itemId, playerMessage?)` | POST | `/api/dialogue/show-item`（SSE） | ✅ v2 |
| `exitDialogue(sessionId, npcId)` | POST | `/api/dialogue/exit` | ✅ |
| `evaluateEnding(sessionId)` | POST | `/api/game/{id}/evaluate` | ✅ |
| `getItems(sessionId)` | GET | `/api/game/{id}/items` | ✅ v2 |
| `getSessions()` | GET | `/api/sessions` | ✅ |
| `deleteSession(sessionId)` | DELETE | `/api/game/{session_id}` | ✅ |
| `getDialogues(sessionId, npcId?, page?, pageSize?)` | GET | `/api/game/{id}/dialogues` | ✅ |
| `getRelationships(sessionId, npcId?)` | GET | `/api/game/{id}/relationships` | ✅ |
| `getEvents(sessionId)` | GET | `/api/game/{id}/events` | ✅ |
| `parseSSEStream(stream, callbacks)` | - | SSE 事件解析器 | ✅ |
| `saveGameState(sessionId, state)` | - | localStorage 持久化 | ✅ |

> v2 新增：章节系统 API（`getScripts`/`startChapter`/`getChapter`/`getTask`）、物品系统（`getItems`/`showItemToNpcStream`），以及本地存档槽位管理（`getSaveSlots`/`saveToSlot`/`loadFromSlot`/`deleteSlot`）。所有 API 函数均实现 Mock 数据 + 真实 HTTP 调用双模式。

当前 `USE_MOCK = true`，所有 API 调用走 Mock 数据；`setUseMock(false)` 后切换到真实后端。

## 六、风险与改进建议

### ✅ 已解决
1. **UIScene 完整实现**：对话 UI、SSE 流式显示、选项按钮、文本分页、阶段过渡动画、结局画面、历史面板、ESC 关闭
2. **SSE 解析逻辑**：`parseSSEStream` 回调驱动，正确解析 delta/done/error 三种事件
3. **退出对话 API**：`closeDialog()` 调用 `POST /api/dialogue/exit` 获取 NPC 告别语（fire-and-forget）
4. **存档管理面板**：`MenuScene` 集成 `GET /api/sessions` 列表 + `DELETE /api/game/{id}` 删除，支持滚轮滚动
5. **错误处理增强**：统一 `buildErrorMessage()` 分类处理 HTTP 错误码、网络错误、会话失效等场景
6. **自由文本输入**：DOM `<input>` 元素 + `onOptionSelected` 复用，Enter 发送走 SSE 流式对话
7. **对话框溢出修复**：文本 wordWrap / 选项按钮宽度 / 遮罩全部以面板 `(panelX,panelY,panelW,panelH)` 为唯一基准
8. **历史面板数据源**：H 键调用后端 `getDialogues()` 获取跨轮次完整对话，`lastSpeaker` 去重标签
9. **v2 章节系统**：`startChapter`/`getChapter`/`getTask` API 接入，子任务追踪 + NPC 对话完成度驱动章节推进
10. **v2 全局输入框**：`createGlobalInput()` 工厂函数统一管理 DOM `<input>`，UIScene 复用
11. **v2 本地存档槽位**：6 槽位快捷存取（数字键 1-6 存档，Ctrl+数字键读档）
12. **v2 物品系统**：`getItems`/`showItemToNpcStream` 接入，支持向 NPC 展示物品触发特殊对话
13. **NPC 动态创建**：`refreshNPCsFromState()` 从 API 状态动态生成 5 种 NPC，支持运行时新增/隐藏
14. **六章节色调**：`STAGE_TONES` 扩展至 6 章节 + `CHAPTER_MAP` 映射，`tintOverlay` 自动应用

2. 关系值/事件时间线的可视化展示未接入 HUD
