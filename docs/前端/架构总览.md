## 二、架构模式

### Phaser Scene 场景驱动架构

| 层级 | 对应概念 | 说明 |
|------|---------|------|
| **Scene 层** | 类似"视图层" | 每个 Scene 管理一组完整的游戏画面和逻辑 |
| **Event Emitter 层** | 类似"消息总线" | Scene 间通过 `scene.events` 进行解耦通信 |
| **DOM 辅助层** | 原生输入覆盖 | `index.html` 中 `<input>` 等 DOM 元素，由 Scene 控制显隐，用于自由文本输入 |
| **API 层** | 数据访问层 | `src/api/client.js` 封装所有后端通信（28 个 API 函数 + SSE 解析器，Mock + 真实双模式，v3 新增持久化存档管理） |
| **Config 层** | 常量配置 | `src/config.js` 存放游戏常量、配色、场景 key |

## 三、Scene 分析

### 3.1 main.js — 游戏入口

```javascript
const config = {
  type: Phaser.AUTO,
  parent: 'game-container',
  width: 1280, height: 800,
  backgroundColor: '#1a1a2e',
  pixelArt: true,
  roundPixels: true,
  dom: { createContainer: true },              // ← 启用 DOM 元素层
  scale: { mode: Phaser.Scale.RESIZE, autoCenter: Phaser.Scale.CENTER_BOTH },
  physics: { default: 'arcade', arcade: { gravity: { x: 0, y: 0 }, debug: false } },
  scene: [BootScene, MenuScene, GameScene, UIScene],
};
```

同时导出全局自由文本输入工厂函数 `createGlobalInput()` 和 `globalInputValues`，供 UIScene 在对话中显示 DOM `<input>` 元素实现自由文本输入。

### 3.2 BootScene — 启动场景

最小化启动器，`preload()` 为空，`create()` 直接转入 MenuScene。

### 3.3 MenuScene — 主菜单 + 存档管理

提供"开始游戏"和"继续游戏"两个核心按钮。点击"继续游戏"直接弹出存档列表面板（`showArchivePanel()`），调用 `GET /api/sessions` 从后端获取所有会话，渲染带"继续"/"删除"按钮的卡片式列表。面板支持滚轮滚动、ESC 关闭、遮罩点击关闭。

### 3.4 GameScene — 核心游戏场景

GameScene 作为游戏世界的协调器，将地图生成、碰撞编辑、NPC 管理等独立功能委托给 `modules/` 下的子模块：

- **MapGenerator**: 程序化生成 80×50 格小镇地图（含建筑、道路、河流）
- **CollisionEditor**: E 键进入的可视化碰撞格子编辑器（涂画/擦除/NPC拖拽/WASD视角）
- **GameUIHelpers**: 共享的精灵配置常量（主角/NPC）、Toast/Loading 提示工具

#### 地图系统
- **预加载大图贴图**：使用 `town_worldmap.png` 作为地图背景（`/assets/images/maps/town_worldmap.png`）
- 地图放大 `MAP_SCALE = 1.8`，支持摄像机平滑滚动
- 地图尺寸保持 80×50 格 × 16px 的虚拟坐标体系（用于碰撞编辑器和 NPC 定位）
- **程序化生成保留**（`generateMapData()` 函数仍存在，用于将来）

#### 碰撞编辑器
- 按 **E** 键进入编辑模式：WASD 自由滚动视角，鼠标左键切换格子碰撞状态，拖拽移动 NPC 位置
- 按 **S** 保存（编辑器内），按 **C** 清除所有碰撞，按 **R** 硬重置（仅清存档，保留编辑配置）
- 碰撞数据保存到 `localStorage.editor_collision_map`，编辑器网格以红色高亮碰撞格、青色标记 NPC
- NPC 拖拽到新位置后自动吸附到格子中心，位置保存到 `localStorage.editor_npc_positions`
- 玩家移动时通过 `_checkCollisionAt()` 检测前方 2×2 区域是否有碰撞格（探测距离 8px）

#### 玩家系统
- **真实精灵图**：使用 `/assets/images/characters/protagonist/sprites/` 下的 8 张单帧图片（4方向 × idle/walk）
- 主角配置 `PROTAGONIST`（baseDir、prefix、scale: 0.09、bodyRatio 碰撞体）
- `switchPlayerTexture(facing, isMoving)` 按朝向+运动状态切换纹理，切换后自动重建 physics body
- WASD + 方向键双模式控制，斜向速度归一化
- 自定义摄像机边界（不依赖 `setCollideWorldBounds`），手动 clamp 防止越界

#### NPC 系统
5 个主要 NPC，全部使用真实精灵图：
| NPC | 精灵前缀 | 精灵目录 | scale |
|-----|---------|---------|-------|
| 陈师傅 (npc_chen) | `chenshifu` | `npc-chenshifu` | 0.09 |
| 小华 (npc_xiaohua) | `xiaohua` | `npc-xiaohua` | 0.09 |
| 老周 (npc_laozhou) | `laozhou` | `npc-laozhou` | 0.09 |
| 梅姨 (npc_meiyi) | `meiyi` | `npc-meiyi` | 0.09 |
| 船夫老李 (npc_laoli) | `chuanfulaoli` | `npc-chuanfulaoli` | 0.09 |

每个 NPC 4 方向 × 2 状态（idle/walk）= 8 张精灵图，通过 `NPC_SPRITES` 配置映射。NPC 通过 `refreshNPCsFromState()` 从后端 API 返回动态生成，支持运行时新增/隐藏/更新。头顶气泡显示 `current_greeting` 问候语。

#### 交互流程（v2 按钮式）
```
player overlap npc → 在 NPC 上方显示交互按钮组：
  [💬 进行对话] → events.emit('dialogue:start', {npcId, name, position})
  [🎁 展示物品] → events.emit('show-item:select', {npcId, npcName})
→ UIScene 监听并弹出对应 UI
```
- NPC 交互不再使用 F 键，改为悬浮按钮（npcActionContainer）
- F 键仅用于物品拾取

#### 场景物品系统
- 从后端 `GET /api/game/{id}/items` 获取 `scene_items` 列表
- `createSceneItemSprite()` 在地图上创建 📦 图标精灵（含浮动动画）
- 玩家接近时显示"按 [F] 拾取 XX"，`pickupItem()` 调用 `POST /api/game/{id}/item/discover`
- 发现后通知 UIScene 刷新背包（`item:discovered` 事件）

### 3.5 UIScene — UI 覆盖层协调器

UIScene 运行于 GameScene 之上，作为各 UI 功能模块的组合器：
- **DialogueManager**: 对话框面板 + SSE 流式显示 + 文本分页 + 选项按钮 + 自由文本输入
- **InventoryPanel**: 左右分栏背包面板 + 展示物品模式（从 NPC 交互触发）
- **HistoryPanel**: 全屏对话历史面板（按章节分组、滚轮滚动）
- **SaveManager**: 暂停菜单 + 6 槽位存档/加载 + 音量滑块
- **EndingScreen**: 结局评价画面（逐行弹出动画）
- **StageTransition**: 章节过渡动画（淡入/淡出 + 标题描述）

#### 对话系统
- **对话面板**（`createDialogPanel`）：底部 210px 高的半透明面板，含 NPC 名字、文本滚动区域（带 GeometryMask 裁剪 + 滚动条）、选项按钮区
- **SSE 流式显示**：`parseSSEStream` 回调驱动，`onDelta` 逐 chunk 追加文本 + 自动滚底，`onDone` 触发分页
- **文本分页**（`splitTextToPages`）：二分查找截断点，超长文本自动分多页显示，支持点击翻页（右下角页码提示）
- **选项按钮**：动态宽度计算，二次校验不超面板边界，鼠标 hover 高亮，数字键 1-3 快捷选择
- **自由文本输入**：对话时底部显示 DOM `<input>`（通过 `createGlobalInput()` 工厂），支持自由输入文字按 Enter 发送，与预设选项共用 `onOptionSelected` 逻辑调用 SSE 流式接口
- **物品展示对话**（v2 新增）：通过 `showItemToNpcStream` 向 NPC 展示背包物品，触发特殊对话

#### 章节系统集成（v2 新增）
- **章节开始**：GameScene 初始化时调用 `startChapter()` 获取当前章节和子任务
- **阶段过渡动画**（`playStageTransition`）同步使用 `STAGE_TONES` 中的章节信息（名称、色彩、描述）
- **子任务追踪**：`getTask()` 获取各 NPC 对话完成度（`npc_completion_votes`）驱动章节推进
- **章节完成流程**：SSE `done` 事件 `chapter_completed=true` → `startChapter()` 推进 → 调用 `gs.collectPositions()` 收集所有 NPC/主角实时位置 → `batchReportNPCPositions()` 上报到后端 → `getGameState()` 刷新最新状态注入位置数据 → `emit('state:refresh', state)` 更新场景

#### 记忆回想（历史面板）
- **全屏历史面板**（`createPanel`）：按 H 键切换，GeometryMask 裁剪内容区，滚轮滚动
- **数据源**：`_restoreDialogueHistory()` 异步调用 `GET /api/game/{id}/dialogues` 获取该 session 完整对话历史，`_extractText()` 递归提取 `dialogue_text`/`text`/`content` 字段，`_resolveNpcName()` 通过场景 NPC 或硬编码映射将 npc_id 转中文名，失败回退 localStorage 缓存
- **标签去重**：`lastName` 变量追踪上一轮说话人，同一 NPC 连续发言不重复显示名字标签
- **窗口缩放**：`onResize()` 销毁重建面板容器，恢复显隐状态 + 滚动位置

#### 存档系统（v3 后端持久化）
- **快捷存档**：游戏中按数字键 1-6 调用 `createSave(sessionId, gameState, slotId)` 存入后端 + localStorage 双写
- **快捷键**：Ctrl+数字键 1-6 调用 `loadSave(sessionId, saveId)` 从后端恢复完整状态
- **槽位管理**：每个 session 最多 6 个槽位，满时自动覆盖最旧的，槽位列表通过 `getSaves(sessionId)` 查询，`deleteSave(sessionId, saveId)` 删除
- **返回主菜单**：`onReturnToMenu()` 中先调用 `collectPositions()` 收集 NPC/主角位置 → `batchReportNPCPositions()` 上报 → 写入 `state._town_npc_positions` + `state._player_position` → `saveGameState()` 持久化 → 跳转

#### 阶段过渡与结局
- **阶段过渡动画**（`playStageTransition`）：全屏深色遮罩 + 章节标题 + 描述淡入，停留 2.2s 后淡出
- **结局画面**（`triggerEndingSequence`）："命运的齿轮开始转动……" → 调用 `POST /api/game/{id}/evaluate` → 逐行显示标题/关键节点/人生感悟/NPC 结局

#### 输入与状态管理
- **ESC 关闭**：对话框翻页中 ESC 跳过翻页，选项时 ESC 关闭对话（调用 `exitDialogue` API 非阻塞获取告别语）
- **自由输入保护**：DOM 输入框聚焦时跳过 Phaser 键盘事件处理
- **错误处理**：
  - API 层：`_extractApiError()` 从 FastAPI 错误响应 `{"detail":{"error":true,"code":"...","message":"..."}}` 中提取人类可读消息，fallback 到 HTTP 状态码
  - UI 层：`catch (err)` 中错误提示动态展示 `${err.message}`，兜底「网络开小差了，请重试」
  - `buildErrorMessage()` 按特定错误码（SESSION_NOT_FOUND / NPC_NOT_FOUND / NETWORK_ERROR）返回中文提示

#### 窗口缩放响应
- **`_onResize()`**：Phaser `scale.on('resize')` 事件回调，100ms 防抖避免初始化时连续触发导致崩溃
- **`_doResize()`** 实际执行：更新右上角 HUD 坐标（stageBadge/historyBtn/backpackBtn），重定位自由输入框 DOM，按优先级顺序遍历 `[dialogue, inventoryPanel, historyPanel, saveManager, stageTransition, endingScreen]` 各模块的 `onResize()`，每个模块独立 try-catch 防止连锁崩溃
- **HistoryPanel.onResize()**：销毁重建 `_historyPanelUI`，恢复显隐状态 + `historyScrollY` 滚动位置
- **InventoryPanel.onResize()**：销毁重建 `backpackPanel`，恢复显隐/展示模式/游标索引，刷新内容+高亮
- **DialogueManager.onResize()**：销毁重建对话面板，恢复对话文本+选项+分页状态

## 四、数据流向

```
[用户操作]
    │
GameScene.update()              ← Phaser 游戏循环（60fps）
    │
    ├── WASD → player.setVelocity()
    ├── overlap 检测 → 显示 NPC 交互按钮(💬对话/🎁展示物品) + 物品 [F] 提示
    └── 按钮点击 / JustDown(F) → emit('dialogue:start') / pickupItem()
                           │
                     UIScene (监听事件)
                           │
              ┌────────────┼────────────┐
              │            │            │
      预设选项按钮    自由文本输入    H键历史面板
      (数字键+点击)  (Enter发送)   (全屏记忆回想)
              │            │            │
              └────────────┼────────────┘
                           │
                     api/client.js
           ┌───────────────┼───────────────┐
           │               │               │
    startDialogueStream  getDialogues   exitDialogue
    (POST /api/dialogue  (GET /api/game/ (POST /api/dialogue
     → SSE 流式)          {id}/dialogues) /exit)
    showItemToNpcStream  getItems        evaluateEnding
    (POST /api/dialogue  (GET /api/game/ (POST /api/game/
     /show-item → SSE)   {id}/items)     {id}/evaluate)
    discoverItem
    (POST /api/game/
     {id}/item/discover)
           │
    parseSSEStream
    ├── onDelta(chunk) → dialogText 逐字追加
    ├── onDone(result) → 分页 + 选项 + 章节切换/结局
    └── onError(err)   → 动态展示 err.message（由 _extractApiError 提取）→ 兜底"网络开小差了"

[章节流程 — v2 新增]
GameScene.initGame() → startChapter()
    │
    └── 返回 task (sub_tasks + npc_completion_votes)
         → UIScene 根据 task 更新 UI
         → 各 NPC 对话完成后推进章节

[存档管理]
MenuScene.onContinue() → showArchivePanel()
    │
    ├── getSessions()     → GET /api/sessions → 渲染存档卡片列表
    ├── deleteSession(id) → DELETE /api/game/{id} → 删除 + 刷新列表
    └── loadArchive(id)   → scene.start('GameScene', {savedSessionId: id})
                              → GameScene.restoreGame(id)
                                  ├── getGameState(id) → GET /api/game/{id}
                                  ├── startChapter(id) → 恢复章节上下文
                                  └── 失败回退 localStorage

[本地存档槽位 — v2 新增]
游戏中按数字键 1-6
    ├── saveToSlot(sessionId, gameState, slotId) → localStorage 6 槽位
    └── 按 Ctrl+数字键 → loadFromSlot(slotId) → 恢复游戏
```

## 五、API 客户端

> 📎 **详见 [后端/API设计文档.md](../后端/API设计文档.md)** — 完整的接口请求/响应格式。

`api/client.js`（v2 章节驱动版）暴露以下函数。Mock 数据和 SSE 解析器已拆分为独立模块 `mock-data.js` 和 `sse-parser.js`。

| 函数 | HTTP | 路由 | 状态 |
|------|------|------|------|
| `startGame(playerName, scriptId)` | POST | `/api/game/start` | ✅ |
| `getGameState(sessionId)` | GET | `/api/game/{id}` | ✅ |
