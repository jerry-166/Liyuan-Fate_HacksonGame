## 二、架构模式

### Phaser Scene 场景驱动架构

| 层级 | 对应概念 | 说明 |
|------|---------|------|
| **Scene 层** | 类似"视图层" | 每个 Scene 管理一组完整的游戏画面和逻辑 |
| **Event Emitter 层** | 类似"消息总线" | Scene 间通过 `scene.events` 进行解耦通信 |
| **API 层** | 数据访问层 | `src/api/client.js` 封装所有后端通信 |
| **Config 层** | 常量配置 | `src/config.js` 存放游戏常量、配色、场景 key |

### 目录结构

```
frontend/
├── index.html             # 入口 HTML
├── package.json           # 依赖：仅 phaser + vite
├── vite.config.js         # Vite 配置（proxy /api 到 8000）
└── src/
    ├── main.js            # 游戏启动入口，Phaser.Game 实例化
    ├── config.js          # 全局常量（尺寸、颜色、场景 key、阶段色调）
    ├── api/
    │   └── client.js      # API 客户端（Mock 数据 + 真实 HTTP 调用）
    └── scenes/
        ├── BootScene.js   # 启动场景（空加载，直接转 GameScene）
        ├── GameScene.js   # 核心游戏场景（地图、玩家、NPC、交互）
        └── UIScene.js     # UI 覆盖层场景（TODO：对话 UI / HUD）
```

## 三、Scene 分析

### 3.1 main.js — 游戏入口

```javascript
const config = {
  type: Phaser.AUTO,
  parent: 'game-container',
  width: 1024, height: 768,
  backgroundColor: '#1a1a2e',
  pixelArt: true,
  roundPixels: true,
  scale: { mode: Phaser.Scale.FIT, autoCenter: Phaser.Scale.CENTER_BOTH },
  physics: { default: 'arcade', arcade: { gravity: { x: 0, y: 0 }, debug: false } },
  scene: [BootScene, GameScene, UIScene],
};
```

### 3.2 BootScene — 启动场景

最小化启动器，`preload()` 为空，`create()` 直接转入 GameScene。当前所有地图、角色、NPC 都是代码绘制（Graphics API），无法美术资源。

### 3.3 GameScene — 核心游戏场景（640 行）

#### 地图系统
- **程序化生成**（非 Tiled tilemap）：80×50 格子，每格 16px = 1280×800 世界
- 12 种 Tile 类型（草地、青石板路、土路、河水、石桥、戏台、茶馆、码头、祠堂、民居、树木）
- 每种 Tile 有 3 色调色板，配合棋盘格实现视觉变化
- 程序化布局：主街、小河、桥、戏台区域、茶馆、码头、祠堂、散落民居

#### 碰撞系统
所有建筑物和自然障碍不可穿越（`GRASS` / `ROAD` / `DIRT` / `BRIDGE` 可通过）。子系统区域入口（茶馆内部等）尚无切换逻辑。

#### 玩家系统
- 程序化绘制 14×28 像素 Q 版角色
- WASD + 方向键双模式控制，斜向速度归一化
- `setCollideWorldBounds(true)` + Arcade Physics collider

#### NPC 系统
2 个 NPC：陈师傅（茶褐色长衫、白发）+ 小华（蓝色短衫、黑发）。头顶气泡显示问候语，物理 `overlap` 检测玩家接近（64px 范围），接近时显示"按 [F] 与 XX 对话"。

#### 交互流程
```
player overlap npc → 显示 [F] 提示 → JustDown(F)
→ events.emit('dialogue:start', {npcId, name})
→ UIScene 监听并弹出对话 UI
```

### 3.4 UIScene — UI 覆盖层（架构占位）

**当前状态：纯占位**——`create()` 为空。这是前端待实现的核心缺口。

**预期职责**：
- 监听 `dialogue:start` 事件 → 弹出对话面板
- SSE 流式展示 NPC 回复文本
- 渲染 AI 生成的对话选项按钮
- 处理阶段切换时的 UI 过渡动画
- 展示结局画面

## 四、数据流向

```
[用户操作]
    │
GameScene.update()          ← Phaser 游戏循环（60fps）
    │
    ├── WASD → player.setVelocity()
    ├── overlap 检测 → 显示 [F] 提示
    └── JustDown(F) → emit('dialogue:start')
                           │
                     UIScene (监听事件)
                           │
                     api/client.js
                     ├── startGame()          → POST /api/game/start
                     ├── getGameState()       → GET /api/game/{id}
                     ├── startDialogueStream() → POST /api/dialogue → SSE
                     └── evaluateEnding()     → POST /api/game/{id}/evaluate
```

> 📎 **SSE 通信格式详见 [_shared/SSE通信格式.md](../_shared/SSE通信格式.md)**

## 五、API 客户端

> 📎 **详见 [后端/API设计文档.md](../后端/API设计文档.md)** — 完整的接口请求/响应格式。

当前 `api/client.js` 暴露以下函数：

| 函数 | HTTP | 路由 |
|------|------|------|
| `startGame(playerName)` | POST | `/api/game/start` |
| `getGameState(sessionId)` | GET | `/api/game/{id}` |
| `startDialogueStream(...)` | POST | `/api/dialogue`（SSE） |
| `evaluateEnding(sessionId)` | POST | `/api/game/{id}/evaluate` |

当前 `USE_MOCK = true`，所有 API 调用走 Mock 数据；`setUseMock(false)` 后切换到真实后端。

## 六、风险与改进建议

### 🔴 高风险
1. **UIScene 空白**：对话 UI 是 MVP 核心交互，尚未开始
2. **无 SSE 解析逻辑**：`startDialogueStream` 返回了 `ReadableStream`，但无任何代码消费它

### 🟡 中风险
3. 缺少对 `stage_changed` / `relationship_change` / `events_triggered` 的事件响应逻辑
4. 无错误处理（try-catch 和 error SSE 事件均未处理）
5. 无子场景切换（茶馆内部、戏台后台）

### 🟢 低风险
6. NPC 头顶气泡文本硬编码，未消费后端 `current_greeting`
7. 音箱/音乐尚未实现
