# 前端启动指南

---

## 前置条件

- Node.js >= 16
- npm（随 Node.js 安装）

## 安装依赖

```powershell
cd D:\workspace\A_Hackson_game\frontend

# 安装依赖
npm install
```

## 启动方式

### 开发模式

```powershell
npm run dev
```

服务启动在 **http://localhost:3000**，自动打开浏览器。

### Mock 模式（默认，可脱离后端独立运行）

默认使用 `VITE_USE_MOCK=true`，所有 API 调用走 `src/api/client.js` 中的内联 Mock 数据（v2 章节驱动版）：

| 接口 | Mock 场景 |
|------|----------|
| `POST /api/game/start` | 返回固定初始游戏状态（含 5 NPC、6 章节参数） |
| `GET /api/game/{id}` | 返回会话完整状态 |
| `POST /api/dialogue` | 按对话轮次自动切换 7 种 SSE 场景（首轮多NPC开场/续接/章节完成/结局触发/无选项） |
| `POST /api/game/{id}/evaluate` | 返回预设结局 JSON（含关键节点、NPC 结局） |
| `GET /api/scripts` | 返回剧本列表 |
| `POST /api/game/{id}/chapter/start` | 返回当前章节任务（含子任务、NPC 完成度投票） |
| `GET /api/game/{id}/items` | 返回物品列表 |
| `POST /api/dialogue/show-item` | 展示物品给 NPC，触发特殊对话 |

切换到真实后端只需在启动时设置环境变量：

```powershell
# PowerShell
$env:VITE_USE_MOCK = "false"
npm run dev

# 或直接修改 client.js 中的默认值（约第 224 行 let USE_MOCK = ...）
```

### 生产构建

```powershell
npm run build
```

产出在 `frontend/dist/` 目录，可直接部署到任何静态服务器。

### 预览构建产物

```powershell
npm run preview
```

## 与后端联调

### 自动代理

`vite.config.js` 已配置 `/api` 代理到 `http://localhost:8000`：

```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  }
}
```

因此后端启动后（默认 8000 端口），前端不需要任何额外配置即可联调。

### 联调步骤

```
1. 启动后端（8000）        → 见 docs/后端/启动指南.md
2. 启动前端（3000）        → npm run dev
3. 浏览器打开 localhost:3000
4. 前端 API 调用 /api/xxx  ← 自动代理到后端
```

### 常见问题

**Q: 请求报 404 / 连接拒绝？**
A: 检查后端是否在 8000 端口运行。确认 `vite.config.js` 的 proxy target 与后端地址一致。

**Q: 想从 Mock 切换到真实后端失败了？**
A: 确保方式正确——要么设环境变量 `VITE_USE_MOCK=false` 再启动，要么在运行时调用 `setUseMock(false)`。修改 `.env` 文件不生效，因为项目未引入 dotenv。

**Q: `npm install` 报权限错误？**
A: Windows 下以非管理员身份运行时，尝试 `npm install --no-optional` 跳过可选依赖。

## 切换 Mock/真实模式

| 方式 | 作用域 | 方法 |
|------|--------|------|
| 环境变量 | 全局（推荐） | 启动前 `$env:VITE_USE_MOCK = "false"` |
| 函数调用 | 运行时动态 | `import { setUseMock } from './api/client.js'; setUseMock(false)` |
| 修改源码 | 永久 | 编辑 `client.js` `let USE_MOCK = ...` 行 |
