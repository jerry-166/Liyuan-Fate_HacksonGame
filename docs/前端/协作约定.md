| # | 前端当前状态 | 对人 B 意味着什么 | 优先级 |
|---|------------|-----------------|--------|
| 1 | `UIScene` 已完整实现（对话/SSE/选项/自由输入/阶段过渡/结局/历史面板/章节系统/物品展示/存档/任务面板/剧本面板/BGM） <br> `GameScene` 新增子场景系统（6室内场景进入/离开/世界状态保存恢复/碰撞隔离） | B 的 SSE 流式结构和 API 响应字段已对接完成 | 🟢 P2 |
| 2 | `api/client.js` 29+ 个 API 函数全量接入（v3，含章节/物品/NPC位置/普通NPC管理/持久化存档，Mock + 真实双模式） | B 确保真实 API 与 Mock 数据结构完全对齐 | 🟡 P1 |
| 3 | 六章节色调 + `CHAPTER_MAP` 已定义，`tintOverlay` 自动应用 | B 在 SSE `done` 事件中准确返回 `current_chapter` + `chapter_completed` | 🟡 P1 |
| 4 | 对话历史存储在后端 SQLite | B 的状态管理器支持 `GET /api/game/{id}` 返回完整状态 | 🟡 P1 |
| 5 | NPC 气泡文本从 API 状态 `current_greeting` 动态更新（`refreshNPCsFromState`） | B 的 `POST /api/game/start` 及状态接口返回准确 `current_greeting` | 🟢 P2 |
| 6 | SSE data 行解析依赖格式一致性 | B 确保 SSE data 行是单行紧凑 JSON | 🟡 P1 |
| 7 | 自由文本输入框为 DOM `<input>`（`createGlobalInput()` 工厂） | B 的 `POST /api/dialogue` 支持 `player_message` 字段即可复用 | 🟢 P2 |
| 8 | 章节系统：`startChapter`/`getChapter`/`getTask` 已接入 | B 的章节 API 返回 `task.sub_tasks` + `npc_completion_votes` 结构 | 🟡 P1 |
| 9 | 物品展示对话：`showItemToNpcStream` + `getItems` 已接入 | B 的 `/api/dialogue/show-item` 需支持物品上下文对话 | 🟢 P2 |
| 10 | 本地存档槽位：数字键 1-6 存档，Ctrl+数字键读档 | B 的 `GET /api/game/{id}` 需支持从槽位恢复完整状态 | 🟢 P2 |
| 11 | API 错误响应采用 FastAPI 格式 `{"detail":{"error":true,"code":"...","message":"..."}}`，前端 `_extractApiError()` 从 `detail.message` 提取消息 | B 确保所有 HTTP 4xx/5xx 响应的 `detail.message` 为面向用户的中文提示，避免返回技术细节 | 🟡 P1 |
| 12 | **命名对话框**：MenuScene 中玩家命名使用严格模态弹窗（DOM input + Phaser 遮罩），支持空名称校验 + shake 动画 | B 的 `POST /api/game/start` 接受 `playerName` 参数（1-12 字符）即可 | 🟢 P2 |
| 13 | **剧本工坊（EditorPanel）**：DOM iframe 覆盖层加载 `editor.html`，通过 postMessage 双向通信 | B 的编辑器脚本支持 iframe 嵌入模式（`embedded=1` query param） | 🟡 P1 |
| 14 | **世界资产浏览**：工坊内新增物品/场景浏览视图，调用 `GET /api/world/items` 和 `GET /api/world/scenes` | B 确保两个世界 API 返回稳定的 JSON 格式（items 含模板合并 + 场景含连接拓扑） | 🟡 P1 |
