# 前端协作约定

> 本文档记录前端与后端（人 B）、内容/美术（人 C）之间的协作交叉点。

---

## 与后端（人 B）的协作点

| # | 前端当前状态 | 对人 B 意味着什么 | 优先级 |
|---|------------|-----------------|--------|
| 1 | `UIScene` 是空的 `create() {}` | B 需要尽快提供 SSE 解析和对话 UI 的规范 | 🔴 P0 |
| 2 | `api/client.js` 使用 `USE_MOCK` toggle | B 确保真实 API 与 Mock 数据结构完全对齐 | 🟡 P1 |
| 3 | 阶段色调已定义但未使用 | B 在 SSE `done` 事件中准确返回 `stage_changed` + `new_stage` | 🟡 P1 |
| 4 | 对话历史存储在后端 SQLite | B 的状态管理器支持 `GET /api/game/{id}` 返回完整状态 | 🟡 P1 |
| 5 | NPC 头顶气泡文本硬编码 | B 的 `POST /api/game/start` 返回 `current_greeting` 作为前端更新源 | 🟢 P2 |
| 6 | SSE data 行解析依赖格式一致性 | B 确保 SSE data 行是单行紧凑 JSON | 🟡 P1 |

## 与内容/美术（人 C）的协作点

| # | 前端当前状态 | 对人 C 意味着什么 | 优先级 |
|---|------------|-----------------|--------|
| 1 | 所有角色和地图是程序化绘制，无外部图片 | C 提供真实的 tile 贴图 + NPC sprites | 🔴 P0 |
| 2 | NPC 硬编码 2 个（陈师傅、小华） | C 的人设卡直接影响 NPC 外观设计 | 🟡 P1 |
| 3 | NPC 初始问候语硬编码 | C 提供更丰富的对话库 | 🟢 P2 |
| 4 | 地图布局与 C 的"关键地点"对齐 | C 基于此布局设计更精细的 tile 贴图 | 🟡 P1 |
| 5 | 三阶段色调定义（冷→暖→戏剧性）与 C 的叙事三阶段对齐 | C 提供色调指导（和可能的背景替换图） | 🟢 P2 |
| 6 | 对话选项目前只有 Mock | C 设计关键剧情节点和结局 | 🟡 P1 |
| 7 | 音箱/音乐未实现 | C 的 AI 音乐产出需与阶段色调一一对应 | 🟢 P2 |

## 综合协作状态

```
前端 (人A)          后端 (人B)            内容 (人C)
──────────────────────────────────────────────────
✅ 地图渲染          🔴 后端代码进行中    🔴 NPC人设卡未交付
✅ WASD控制          🟡 API 已设计        🔴 tile/角色图未交付
🔴 UIScene空白       ✅ Mock 数据已提供    🟡 微剧本大纲进行中
🟡 对话API封装完成   🟡 架构方案完成       🟡 音乐/音效未启动
```

> **MVP 关键路径**：A 实现 UIScene(对话UI) ← 依赖 → B 完成后端核心链路、C 提供 NPC 人设 + 基础美术。三者需要同步推进。
